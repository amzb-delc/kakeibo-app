家計簿アプリ — MVP仕様書

最終更新: 2026-05-21（モーダル化 + 開閉アニメ/下スワイプ + フォーム刷新・カテゴリ色分け反映）

概要

夫婦2人で日々の支出を素早く記録し、月単位で振り返るための家計簿アプリ。iPhone Safari をメインデバイスとして PWA 対応する。

項目内容
利用者夫婦2人（1世帯）
主要デバイスiPhone（Safari、PWAインストール想定）
目的支出の即時記録 + 月次の使い方の可視化
認証MVPでは未実装。固定の demo-household にスコープ
タイムゾーンJST（Asia/Tokyo）固定

技術スタック

レイヤー技術
フレームワークNext.js 16（App Router、Turbopack dev）
言語TypeScript
UIshadcn-ui + Tailwind CSS
DBPostgreSQL
ORMPrisma 5（src/lib/prisma.ts でシングルトン）
PWApublic/manifest.json、アイコン PNG（public/character.png から生成）

画面構成

/ ホーム

当月支出のサマリーカード。左にキャラクター（public/character.png）を配置。

┌─────────────────────────┐ │ ホーム │ │ ┌───────────────────┐ │ │ │ (^o^) 5月の支出 │ │ ← 左にキャラ（72px） │ │ 👽 ¥142,500 ↑+8%│ │ │ │ 先月:¥131,800│ │ │ └───────────────────┘ │ └─────────────────────────┘

データ: 当月合計、前月合計、トレンドアイコン、前月比%（前月0円の場合は非表示）。サーバコンポーネントで Prisma 直接取得。

空状態: 当月合計が 0 のときはキャラを大きく中央配置（128/160px）し、「{month}月の記録はまだないよ」「右下の ＋ から登録してね」を表示。

/summary 月次サマリー

月切り替え（◀▶）と、カテゴリ別の支出可視化。当月閲覧時は比較表示なし、過去月閲覧時は「今月」との差額を表示する（今月視点：今月が表示月より多ければ赤、少なければ緑）。

項目仕様
月切り替えクライアント側 useState で year/month を保持、API再取得
翌月ボタン当月なら disabled
合計カードドーナツチャート（src/components/donut-chart.tsx、自前 SVG、max 180px / ringWidth 22）の中央に「合計」ラベル → 表示月合計（text-xl）→ 差額（過去月閲覧時のみ、トレンド色）を配置。ドーナツのセグメントは金額の大きい順（降順）、カテゴリ色で塗り。カード右上に上位3カテゴリのレジェンド（カテゴリ色のドット＋カテゴリ名のみ、%なし）。合計カードの異常値バーは廃止（カテゴリ行のバーは継続）
カテゴリ行カテゴリ名は色分けタグ（category-color.ts）。右側は金額の下に差額（過去月閲覧時のみ、トレンド色）の縦並び。下に異常値バー（フィル長＝支出量、バー色はカテゴリ色）。タップで展開（aria-expanded、可変高さアニメーション）。展開状態はサマリーページが保持（制御コンポーネント）
異常値バー過去6ヶ月（表示月含む）の box stats から「いつもの幅」を算出し、上フェンス（Q3+1.5×IQR）を 100% としてフィル長で支出量を可視化。やたら使った月＝バーが長い、やたら使わなかった月＝バーが短い。サンプル不足（N<2 or IQR=0）の月は斜めストライプ表示（「判定なし」を視覚的に区別、aria-label に明示）。値ありの月は role="progressbar" + aria-valuenow を付与。カテゴリの異常値サンプルはそのカテゴリの支出があった月のみから採る（ゼロ埋めしない）。初回マウントでも 0%→target にトランジションするよう requestAnimationFrame で1フレーム遅延
展開時そのカテゴリの支出明細を日付降順で一行表示（日付 M/D(曜) ／ 店名 or メモ ／ 金額）。明細は monthly-summary API が各カテゴリに同梱するため追加通信なし
明細タップ編集用ボトムシートを開く（useExpenseModal().openEdit(...)）
更新反映登録・編集・削除の成功で ExpenseModalProvider の mutationVersion が増分し、サマリーを自動再取得。画面遷移しないため選択月・展開カテゴリは保持される
空状態カテゴリ件数が 0 のときはキャラクター（public/character.png、112px）を中央に置き、「キロクナシ」を表示（ワレワレ宇宙人のカタコト感に合わせたコピー）

支出モーダル（登録・編集）

支出の登録・編集・削除は専用ページではなくボトムシートのモーダルで行う。ExpenseModalProvider（src/components/expense-modal.tsx）が layout.tsx で全体を包み、FAB・サマリー明細のどちらからでも開ける。画面遷移しないため、背面のサマリー（選択月・展開カテゴリ）が保持される。

挙動

項目仕様
表示fixed inset-0 z-50、下端 anchor（items-end）、最大高 90vh でスクロール、ヘッダは sticky
開閉アニメバックドロップ（dim）はパネルと別レイヤー。パネルは translateY(0 ⇄ 100%) を transition で駆動（cubic-bezier(0.32,0.72,0,1) / 320ms）、dim は opacity を独立フェード。開く時は2フレーム後に表示状態へ切替えて transition を発火。閉じ切ったら DOM から除去（reduced-motion でも timeout で確実に teardown）
フォーム構成並び順は 日付 → カテゴリ → 金額 → 店名 → メモ。全項目ラベルなし（プレースホルダで補助、文字色は text-muted-foreground/60 と薄め）
未入力の強調必須（カテゴリ・金額）が空のとき枠を強調（border-primary/70 ring-1 ring-primary/25）。入力すると解除
日付年月は固定表示（登録=元ページの選択月／編集=対象支出の月）。日のみ縦ホイール（day-wheel.tsx）で選択し、「YYYY年M月 d 日」の d にインライン埋め込み。spentAt は YYYY-MM-DD に組み立て
カテゴリSelect（プルダウン）。画面外要素の誤選択を防ぐためチップ横スクロールから変更。色分けタグで表示。起動時に GET /api/categories を先読みキャッシュ。初期値は setComposeContext の展開カテゴリ
金額¥プレフィクス・右寄せ。システムの数値キーボード（inputMode="numeric"、数字のみ・最大9桁）。カテゴリと横並び・同じ高さ 48px。0円も可（未入力のみ不可）
店名 / メモ任意（空文字は null）。プレースホルダは「店名」「メモ」。入力時にハードキャップ 店名 32 / メモ 140 字（フォーム幅に収める）。メモは 3行
保存半分幅・中央。必須（金額>0・カテゴリ）が未充足なら disabled（グレー）。新規 POST /api/expenses / 編集 PATCH /api/expenses/[id] → onSuccess(message)
削除ヘッダ右上のゴミ箱アイコン（✕ の左）。新規時は disabled（グレー）、編集時のみ有効。確認ダイアログ → DELETE /api/expenses/[id]。削除処理は Provider 側が保持（フォームは登録/更新のみ）
成功後親（Provider）がシートを閉じ、共通トースト（「保存/更新/削除しました」）を表示し、mutationVersion を増分
閉じる暗幕タップ / ✕ボタン / Esc / 下スワイプ。開いている間は背面スクロールをロック
下スワイプヘッダ（グラバー）起点の pointer ドラッグでパネルが指追従。110px 超または速い下フリック（>0.5px/ms）で閉じ、未満は 0 へスナップバック。本文スクロールと混線しないようハンドルは touch-none、✕ 上では発火しない

API仕様

すべてのエンドポイントは DEMO_HOUSEHOLD_ID に暗黙スコープされる。認証は未実装。

GET /api/categories

世帯のカテゴリ一覧を sortOrder 昇順で返す。

レスポンス: Category[]

[
  { "id": "...", "name": "食費", "sortOrder": 0 },
  ...
]

GET /api/expenses

クエリで月とカテゴリで絞り込み。最大 500 件、spentAt 降順 → createdAt 降順。

クエリ: year, month, categoryId（いずれも任意）。year と month はペアで JST 月境界に変換される。

レスポンス: Expense[]（category: { id, name } を include）

POST /api/expenses

支出を新規作成。

必須: amount（0〜100,000,000の整数。0円可）、spentAt（YYYY-MM-DD）、categoryId（世帯のカテゴリ）

任意: storeName（≤100）、memo（≤500）、receiptImageUrl

レスポンス: 201 + 作成された Expense / 400 + { error }

{
  "amount": 1280,
  "spentAt": "2026-05-20",
  "categoryId": "...",
  "storeName": "スーパー〇〇",
  "memo": null
}

GET /api/expenses/[id]

1件取得。他世帯のIDは 404。

PATCH /api/expenses/[id]

部分更新。送信したフィールドのみ更新（undefined は無視）。

世帯スコープ: updateMany({ where: { id, householdId } }) で 1 クエリ。0件なら 404。

バリデーション: POSTと同じガード関数（validateExpenseInput(body, { partial: true })）。

空文字の扱い:

フィールドundefined""
amountスキップ400
spentAtスキップ400
categoryIdスキップ400
storeName / memo / receiptImageUrlスキップnull にクリア

DELETE /api/expenses/[id]

削除。deleteMany({ where: { id, householdId } }) で 1 クエリ。0件なら 404。

GET /api/monthly-summary

表示月のカテゴリ別集計と、比較用の「今月」集計、そして直近6ヶ月（表示月含む）の box stats（中央値・四分位・フェンス）を返す。当月閲覧時は compareTotal: null。box stats はサマリーの「異常値バー」のフィル基準として使う。各カテゴリに表示月の支出明細（spentAt 降順）を同梱するため、サマリー画面は追加通信なしで明細を展開できる。

クエリ: year, month（未指定なら当月）

レスポンス（過去月閲覧時）:

{
  "year": 2026,
  "month": 4,
  "total": 131800,
  "compareTotal": 142500,
  "boxStats": {
    "median": 130000,
    "q1": 120000,
    "q3": 145000,
    "lowerFence": 82500,
    "upperFence": 182500
  },
  "categories": [
    {
      "categoryId": "...",
      "name": "食費",
      "sortOrder": 0,
      "total": 48100,
      "compareTotal": 52300,
      "boxStats": { "median": 49000, "q1": 45750, "q3": 51500, "lowerFence": 37125, "upperFence": 60125 },
      "expenses": [
        {
          "id": "...",
          "amount": 1280,
          "spentAt": "2026-04-20T15:00:00.000Z",
          "storeName": "スーパー〇〇",
          "memo": null
        }
      ]
    },

compareTotal は当月閲覧時に null。boxStats はサンプル<2件 or IQR=0 のときに null。box stats のサンプルは「過去6ヶ月のうち合計支出があった月」を使い、カテゴリ別はその月の該当カテゴリ合計を 0 埋めで取る。当月判定はサーバ TZ に依存せず JST で行う（formatJstDate(new Date()) から年月を抽出）。spentAt は ISO 文字列で返り、クライアントで JST 整形する。

ビジネスルール

日付・タイムゾーン

ユーザーが選ぶ「日付」は JST の暦日として解釈する。サーバの実行 TZ に依存しないよう、src/lib/date.ts で集中管理する。

ヘルパ挙動
jstMonthRange(y, m)JST 月初〜翌月初の { gte, lt } を UTC Date で返す（Date.UTC(..., -9, 0, 0)）
formatJstDate(date)Intl.DateTimeFormat("sv-SE", { timeZone: "Asia/Tokyo" }) で YYYY-MM-DD を返す
todayJst()今日の JST 日付（YYYY-MM-DD）
parseJstDate("YYYY-MM-DD")その日の JST 00:00 を UTC で表現した Date
formatJstDateLabel(date)M/D(曜)
画面表示・DB格納・クエリ境界はすべて同じヘルパを通すこと。直接 new Date(year, month-1, 1) 等を書かない。

入力バリデーション

src/lib/expenses.ts の validateExpenseInput(body, { partial }) に集約。POST/PATCH の双方が同じ関数を通る。

フィールドルール
amount整数 / 0以上 / 100,000,000以下 / 有限値（0円可）
spentAt正規表現 /^\d{4}-\d{2}-\d{2}$/
categoryId非空文字列、かつ DB で householdId = DEMO_HOUSEHOLD_ID の Category に存在
storeName≤100文字、空文字は null
memo≤500文字、空文字は null
receiptImageUrl文字列、空文字は null

世帯スコープ（権限）

  • 全クエリの where に householdId: DEMO_HOUSEHOLD_ID を含める
  • POST/PATCH では categoryId の世帯所属を DB レベルで検証する（IDOR防止）
  • UPDATE/DELETE は updateMany / deleteMany で where に householdId を入れる

ExpenseForm / モーダルの状態

フォームは登録・更新のみを担当（削除は Provider 側）。成功時は onSuccess(message) を呼んで親（Provider）にシートのクローズ・トースト・再取得を委譲する（フォーム自身は画面遷移しない）。送信中のレース回避のため、送信中は全項目を disabled にする。

箇所状態挙動
フォームsubmitting送信中は全項目 disabled、保存ボタンが「保存中…」。必須未充足時は保存ボタンが disabled（グレー）
Providerdeleting削除確認ダイアログの「削除する」を押下中。失敗時はトーストで通知
