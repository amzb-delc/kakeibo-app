全体カードの6ヶ月比較（積み上げ棒）— 実装プラン

作成: 2026-06-12。ステータス: ✅ 実装完了（2026-06-12・PR #71）。 ホームの全体カード（月次サマリーカード）を横スライド式の2ペインに拡張し、従来の単月ペイン（ドーナツ＋レジェンド）に加えて直近6ヶ月のカテゴリ別積み上げ棒グラフを見られるようにする。
グラフは自前 SVG（recharts 等は不採用）で、色は既存ドーナツと共通の categoryColor() パレットを使い、カテゴリ単独比較への切替も持つ。夫婦タグフィルタ（全体/♂/♀）は両ペイン共通で、サーバ側絞り込みのためグラフにも反映される。
API は GET /api/monthly-summary のレスポンスに sixMonths 配列を追加（古い月→表示月の6本固定・タグフィルタ適用後）。

Context（なぜ6ヶ月比較を足すか）

全体カードはこれまで表示月単月の内訳（ドーナツ＋レジェンド）と、過去月閲覧時の「今月との差額」しか持っていなかった。月ごとの支出の推移を一目で比べたい（増えているカテゴリ・季節変動・先月との山の違い）というニーズに対し、単月＋差額だけでは弱い。

そこで全体カードを横スライドで切り替わる2ペインにし、片方に従来の単月内訳、もう片方に直近6ヶ月のカテゴリ別積み上げ棒グラフを置く。グラフは表示月や夫婦タグフィルタ・カテゴリの色付けを単月ペインと共通化し、別画面に飛ばさず同じカード内で完結させる。

UI 設計（スライド式2ペイン）

全体カードを横方向の2ペインに分け、横スライド（約300ms）で切替える。タグフィルタ（全体/♂/♀）はカード上部に置き、両ペイン共通。月送り（◀▶・スワイプ）をしても表示モード（単月／6ヶ月）は維持する。

  • 切替は 300ms の横スライド。表示モードの state はカード側が保持し、月送りやタグ変更では失わない。

グラフ設計（自前 SVG・積み上げ棒）

実装は新規コンポーネント src/components/stacked-bar-chart.tsx。recharts 等の外部ライブラリは使わず、既存の donut-chart.tsx と同じ「自前 SVG」パターンで描く。

  • 6本固定: 左＝最も古い月、右端＝表示月。月送りすると6本がスライドして窓が動く。データのない月は total=0 の空棒として詰めて常に6本を保つ。
  • 積み上げ: 各棒はカテゴリ別に sortOrder 昇順で下から積み上げる。色は既存16色パレット categoryColor()（ドーナツと共通）で、レジェンド／ドーナツと同じ色がそのまま棒にも出る。
  • 正規化: 棒の高さは6ヶ月の最大月合計を 100% として相対化する（窓内で一番高い月がフルハイト）。
  • 月ラベル: 年またぎを考慮し先頭のみ年付き（例 '25 12月、以降は 1月…）。
  • 合計額ラベル: 表示月（右端）の棒のみ合計額を数値で出す（他月は出さず賑やかさを抑える）。

カテゴリ単独比較

6ヶ月ペイン内にカテゴリ選択チップを置く。あるカテゴリを選ぶと、6本の棒をそのカテゴリのみの単色棒（積み上げ解除）に切替え、推移を単独で比較できる。

  • 単独表示時の正規化はそのカテゴリの6ヶ月最大額（全体合計ではなく、選択カテゴリの山に合わせてスケール）。
  • 「全体」チップ／同じカテゴリの再タップで積み上げに戻る。
  • この選択は単月ペインのカテゴリ開閉とは独立した state（6ヶ月ペイン専用。互いに干渉しない）。

夫婦タグフィルタの連動

カード上部のタグフィルタ（全体/♂/♀）は両ペイン共通。spouse:N はサーバ側絞り込み（GET /api/monthly-summary?tag=spouse:N）なので、sixMonths も絞り込み後の値で返り、グラフにも反映される（合計・ドーナツ・前月比・明細と一貫）。

API レスポンス拡張（sixMonths）

GET /api/monthly-summary のレスポンスに sixMonths を追加する。古い月→表示月の6本固定で、tag 指定時はそのフィルタ適用後の値。route は世帯の全カテゴリを取得し、各月の byCategory の名前／色解決に使う。

// レスポンスに追加されるフィールド
"sixMonths": [
  {
    "ym": "2025-12",
    "total": 128400,
    "byCategory": [
      { "categoryId": "...", "name": "食費", "sortOrder": 0, "total": 48100 },
      { "categoryId": "...", "name": "日用品", "sortOrder": 1, "total": 12000 }
    ]
  },
  // ... 古い月 → 表示月 の順で 6 要素固定
]
  • 各月オブジェクト: ym: "YYYY-MM" / total: number（その月の合計）/ byCategory: Array<{ categoryId, name, sortOrder, total }>。
  • 並びは常に6要素（最古→表示月）。データのない月も total=0・byCategory 空で詰める。
  • 集計は src/lib/monthly-summary.ts、route は src/app/api/monthly-summary/route.ts。型は src/types/index.ts に追加。

★ 決定事項

① グラフは自前 SVG（recharts 不採用） ✅ 決定

既存の donut-chart.tsx がすでに「自前 SVG」で実装されており、色付け（categoryColor()）・装飾扱い（aria-hidden）・小幅端末対応のパターンが揃っている。積み上げ棒も同じパターンで描けるため、recharts 等の依存追加を回避し、ドーナツとグラフの色・トーンの一貫性を保つ。バンドルサイズと学習コストの観点でも自前が有利。

② 6本固定 ＋ 空月ゼロ詰め ✅ 決定

窓を常に6本に保つことで、月送りしてもレイアウトが伸縮せず推移が読みやすい。データのない月は total=0 の空棒として詰める（穴を空けない）。

③ 6ヶ月ペインのカテゴリ選択は単月ペインと独立 ✅ 決定

単月ペインの「明細を出すカテゴリ選択」と、6ヶ月ペインの「単独比較するカテゴリ選択」は用途が違うため state を分離。互いの選択が干渉しないようにする。

④ 表示モードは月送り・タグ変更をまたいで維持 ✅ 決定

単月／6ヶ月の表示モードはカード側 state で保持し、月送りやタグフィルタ変更では失わない。「6ヶ月を見ている最中に月送りしたら単月に戻された」体験を避ける。

実機フィードバックによる調整（2026-06-12）

初回実装（f671411 時点の設計）を実機で確認したところ、過去月閲覧時の窓位置やシェブロンの被り・ラベルの賑やかさに改善余地が見つかった。以下は実機フィードバックを受けて入れた調整（バックエンド 510a89e、UI 9a9e05f → 55473c9）。当初設計の「6本固定・古い月→表示月＝右端」は当月閲覧時の挙動として残るが、過去月閲覧時はグラフ窓がずれるため上記④の「表示月＝右端」前提は崩れる（強調ロジックを右端固定から表示月一致判定へ変更）。

① グラフ窓のずれ（過去月閲覧時）

表示月が過去月のとき、6ヶ月窓を未来側へ最大3ヶ月ずらす。表示月の棒が右から 2〜4 番目に来ることで、表示月の「先」の月も見えて推移の文脈が掴みやすくなる。

  • 当月閲覧＝表示月が右端（従来どおり）。前月＝右から2番目／前々月＝3番目／3ヶ月以上前＝右から4番目で固定。
  • ずれ量 forwardMonths = min(max(当月との差, 0), 3)。窓が実在の当月を超えることはない（forward ≤ 当月との差のため）。
  • 偏差値（boxStats）用の「表示月含む過去6ヶ月」窓（sixMonthKeys）は据え置き。グラフ専用に chartKeys を新設し、両者を分離した。DB 取得範囲は両窓の和集合（sixMonthKeys の最古 〜 chartKeys の末尾）。
  • API の sixMonths はこのグラフ窓（chartKeys）に基づく（偏差値窓ではない）。集計は chartKeys.map(...)、route で chartKeys／chartEnd を算出して buildMonthlySummary に渡す。

② シェブロンのフローティング化

シェブロン（‹/›）をペイン内レイアウトから外し、全体カードに対する絶対配置のフローティング丸ボタンに変更。カードのボーダーを跨ぐ位置（左右端・縦中央・半分外側）に置き、コンテンツとの被りを解消した（overflow クリップ枠の外＝カード直下に配置）。

  • 単月表示中は右の › のみ、6ヶ月表示中は左の ‹ のみを出すトグル切替。
  • inert なペインの外なので常に操作可能。ペインから旧ボタン枠（w-7）を除去しコンテンツ幅を回復、横スライドのクリップを overflow-x-clip にして縦の見切れを解消。

③ 積み上げは薄い色・単独比較は濃い色

全体（積み上げ）表示時はセグメントを fillOpacity 0.6 で薄くし、密集した積み上げの可読性を上げる。カテゴリ単独比較時は濃い色（fillOpacity=1）のまま。

④ 全6本に合計額ラベル＋表示月の強調

当初は「表示月（右端）の棒のみ合計額」だったが、全6本の棒に合計額ラベルを出すよう変更。窓ずれで表示月が右端とは限らなくなったため、強調は表示月一致判定（selectedYm）に切替えた。

  • 合計額は省略なしのカンマ区切り（実質6桁までのため崩れない）。0円の月は非表示。
  • 表示月の棒は月ラベルを太字＋下線、金額ラベルも太字で強調（右端とは限らない窓ずれに追従）。

⑤ 目盛線はグラフ縦中央に1本

金額の目盛線はグラフ縦中央に1本だけ引く（薄いグレー）。値は正規化基準＝6本中の最大合計の半分で、右端に省略なし表記の目盛値ラベルを添える（当初の上限＋半分の2本構成から簡素化）。

UIブラッシュアップ（2026-06-15）

6ヶ月グラフ（PR #71 でマージ済み）の表示まわりを、操作系と目盛ラベルの2点でさらに磨いた。ステータス: ✅ 実装完了（2026-06-15・コミット d144665・PR 番号未定／feat/chart-toggle-control）。対象は src/components/monthly-summary.tsx と src/components/stacked-bar-chart.tsx（テストは monthly-summary.test.tsx をトグル仕様へ追従）。

① 表示切替: フローティングシェブロン → 左上トグルスイッチ

これまでカード左右端のボーダーを跨ぐフローティングのシェブロン（›/‹）を廃止し、全体カード左上にアイコントグルスイッチを設置した。タグフィルタと同じセグメント様式（アクティブ側を bg-card ＋ shadow-sm でハイライト）で、右上の夫婦タグフィルタと左右に並ぶ。

  • アイコン: 単月＝ドーナツ（lucide ChartPie）／6ヶ月＝棒グラフ（lucide ChartColumnBig）。aria-hidden のアイコンに対し、各ボタンへ aria-label（「単月の内訳」「6ヶ月の比較」）と aria-pressed（cardMode 一致で true）を付与。外枠は role="group" ＋ aria-label="表示の切り替え"。
  • 切替操作は上段左のトグル（cardMode）が一手に担い、横スライド演出（300ms・2ペインの translateX）自体は据え置き。ペイン内のシェブロン枠は除去した。
  • 経緯: 当初＝ペイン辺の縦長シェブロン（2026-06-12 設計）→ 実機調整でフローティング丸ボタン化（2026-06-12）→ 本改修で左上トグルスイッチに集約。フローティングはカード端でのタップ位置が掴みづらく、タグフィルタと操作面が分かれていたため、両者を上段に並べて操作系を1ヶ所にまとめた。

② 目盛線の金額ラベル: 左ガターへ分離＋短縮表記＋niceRound

グラフ縦中央の目盛線（前回調整で1本に簡素化）に添える金額ラベルを、位置・値・表記の3点で作り直した。

  • 値（niceRound）: 従来は「正規化基準＝6本中最大合計の半分」をそのまま表示していたが、中央付近のキリの良い金額に丸めて表示する。niceRound() は与えた値を 1 / 2 / 5 × 10ⁿ のいずれかへ丸める（例: 47,000 → 50,000）。目盛線の y 位置はこの niceRound 値に対応させる。
  • 位置（左ガター分離）: グラフ左に専用ガター（余白帯 AXIS_W=24）を確保し、棒の描画域（plotLeft = PAD_X + AXIS_W / innerW）をその分だけ右に寄せた。ラベルはこのガター内に右寄せで置くため、棒・合計額ラベルと一切重ならない。
  • 表記（短縮）: formatAxisLabel() で、1万以上は「5万」「100万」（万単位は基本整数・小数が出たら1桁）、1万未満はカンマ区切り。狭いガターに収めるための短縮で、棒の上に出す合計額ラベル（省略なしカンマ区切り）とは別系統（合計額は据え置き）。
