# Design.md — 家計簿アプリ デザインガイド

## 前提

- **利用者**: 夫婦2人
- **デバイス**: iPhone（Safari）メイン
- **目的**: 日々の支出をサッと記録し、月の使い方を振り返る

---

## デザイン原則

1. **片手で完結** — 親指が届く範囲に主要アクションを置く
2. **3タップ以内** — 支出登録は開く→入力→保存で完了
3. **一目で分かる** — 数字とトレンドは視線移動なしで把握できるサイズ
4. **余白で整理** — 情報を詰め込まず、セクションを余白で区切る

---

## カラーパレット

現状のモノクロ（OKLch grayscale）をベースに、アクセントカラーを追加する。

| 用途 | トークン名 | 値（案） | 説明 |
|------|-----------|---------|------|
| 背景 | `--background` | `#FAFAF8` | 温かみのあるオフホワイト |
| カード背景 | `--card` | `#FFFFFF` | 純白カード |
| テキスト | `--foreground` | `#1A1A1A` | ほぼ黒 |
| サブテキスト | `--muted-foreground` | `#6B7280` | グレー |
| プライマリ | `--primary` | `#2563EB` | ブルー（信頼・お金） |
| プライマリ弱 | `--primary-light` | `#DBEAFE` | プライマリの薄い背景 |
| 成功 | `--success` | `#16A34A` | 節約・減少 |
| 警告 | `--warning` | `#F59E0B` | やや増加 |
| 危険 | `--destructive` | `#DC2626` | 大幅増加・エラー |
| ボーダー | `--border` | `#E5E7EB` | 薄いグレー |

### ダークモード

MVP では対応しない。iPhone の「外観モード」に関わらず常にライトモードで表示する。
将来対応する場合は CSS 変数の差し替えだけで済む設計にしておく。

## タイポグラフィ

| 用途 | サイズ | ウェイト | 行間 |
|------|--------|---------|------|
| ページタイトル | 20px (`text-xl`) | Bold (700) | 1.4 |
| セクション見出し | 16px (`text-base`) | Semibold (600) | 1.5 |
| 本文・入力値 | 16px (`text-base`) | Normal (400) | 1.5 |
| 金額（大） | 28px (`text-2xl` 〜 `text-3xl`) | Bold (700) | 1.2 |
| 金額（リスト内） | 16px (`text-base`) | Semibold (600) | 1.5 |
| キャプション・日付 | 13px (`text-xs`) | Normal (400) | 1.4 |

> **注意**: iOS Safari ではフォントサイズが 16px 未満の `<input>` にフォーカスすると自動ズームが発生する。
> すべての入力フィールドは `text-base`（16px）以上にすること。

## スペーシング・レイアウト

### グリッド

- 画面幅いっぱい使用（max-width 制限なし — iPhone 幅で最適化）
- 左右パディング: `px-4`（16px）
- セクション間: `py-6`（24px）

### カード

- 角丸: `rounded-2xl`（16px）
- パディング: `p-4`（16px）
- 影: `shadow-sm`（控えめなエレベーション）
- カード間マージン: `gap-3`（12px）

### タッチターゲット

- 最小サイズ: **44×44px**（Apple HIG 準拠）
- ボタン高さ: `h-12`（48px）以上
- リスト行高さ: `min-h-[56px]`
- アイコンボタン: `w-11 h-11`（44px）以上

- 金額表示はやや大きめフォント（`text-lg`）で右寄せ
- レシート撮影ボタンは MVP では非表示にする（OCR 未実装のため）
- 保存成功時はトースト表示 → ホームに戻る

## コンポーネントスタイル

### ボタン

| バリアント | 用途 | スタイル |
|-----------|------|---------|
| primary | 保存、主要アクション | `bg-primary text-white h-14 rounded-xl text-base font-semibold w-full` |
| secondary | 月次サマリーへ、など | `bg-white border border-border h-14 rounded-xl text-base font-medium w-full` |
| ghost | 戻る、キャンセル | `bg-transparent text-muted-foreground` |
| icon | 月切り替え矢印など | `w-11 h-11 rounded-full flex items-center justify-center` |

### 入力フィールド

```
bg-white border border-border rounded-xl px-4 h-12
text-base placeholder:text-muted-foreground
focus:ring-2 focus:ring-primary/30 focus:border-primary
```

- フォーカス時にリングで強調（`ring-2`）
- ラベルはフィールド上部に小さく（`text-sm font-medium mb-1.5`）

### カテゴリチップ

```
未選択: bg-gray-100 text-gray-700 rounded-full px-4 h-10 text-sm
選択中: bg-primary text-white rounded-full px-4 h-10 text-sm font-medium
```

- 横並び: `flex gap-2 overflow-x-auto scrollbar-hide`
- スナップ: `scroll-snap-type: x mandatory` で快適なスクロール

### カード

```
bg-card rounded-2xl p-4 shadow-sm border border-border/50
```

### トースト（保存成功通知）

```
fixed bottom-6 left-4 right-4
bg-foreground text-background rounded-xl px-4 py-3
text-sm font-medium
```
- 2秒後に自動で消える
- 下からスライドイン

## トランジション・アニメーション

| 要素 | アニメーション | 時間 |
|------|-------------|------|
| ボタンタップ | `active:scale-[0.98]` | 即時 |
| カテゴリ展開 | 高さアニメーション | 200ms ease-out |
| トースト | 下からスライドイン→フェードアウト | 300ms / 200ms |
| ページ遷移 | なし（MVP） | — |
| 月切り替え | コンテンツフェード | 150ms |

> 過剰なアニメーションは避ける。操作の結果が明確に伝わる最小限にとどめる。

## iPhone 最適化

### viewport 設定

```html
<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, viewport-fit=cover">
```

### セーフエリア

- 下部パディングに `pb-[env(safe-area-inset-bottom)]` を適用
- ノッチ/Dynamic Island エリアは sticky ヘッダーで自然にカバー

### PWA 対応（推奨）

```html
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style" content="default">
<link rel="apple-touch-icon" href="/icon-192.png">
```

- ホーム画面に追加して「アプリ風」に使えるようにする
- `manifest.json` を用意し、display: standalone に設定

### 入力の最適化

| フィールド | `inputMode` | `type` | 理由 |
|-----------|------------|--------|------|
| 金額 | `numeric` | `text` | テンキーを出しつつカンマ編集可能に |
| 日付 | — | `date` | iOS ネイティブの日付ピッカー使用 |
| 店名 | — | `text` | 通常キーボード |
| メモ | — | `text` | 通常キーボード |

## トレンド表示

既存の `src/lib/trend.ts` のロジックを踏襲しつつ、視覚表現を強化する。

| レベル | アイコン | テキスト色 | バッジ背景 | 意味 |
|--------|---------|----------|-----------|------|
| up2 | `↑↑` | `text-red-600` | `bg-red-50` | 20%以上の増加 |
| up1 | `↑` | `text-orange-500` | `bg-orange-50` | 5〜20%の増加 |
| flat | `→` | `text-gray-400` | `bg-gray-50` | ±5%以内 |
| down | `↓` | `text-emerald-600` | `bg-emerald-50` | 5%以上の減少 |

- 絵文字（🔺🔻）ではなく矢印テキスト + Badge コンポーネントで統一
- 色覚多様性に配慮し、色だけでなく矢印の方向でも区別可能
